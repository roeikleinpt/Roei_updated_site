# דו״ח Code Review, אבטחה ובדיקות

תאריך הבדיקה: 15 ביולי 2026

## תוצאה כללית

האתר עבר סקירת קוד מלאה, הקשחת נגישות וטפסים, עדכון תלויות, הוספת תשתית בדיקות והרצת בדיקות production. לאחר התיקונים אין שגיאות lint או TypeScript, אין פגיעויות ידועות ב־`npm audit`, הייצוא הסטטי תקין וכל בדיקות הדפדפן האוטומטיות עברו.

## מה תוקן

- טופס יצירת הקשר הופרד ללוגיקת ולידציה בדיקה, הוסר מפתח placeholder, נוספה תצורת סביבה, טופלו תשובות HTTP שגויות, נוספו הודעות נגישות ומיקוד לשדה הראשון שנכשל.
- טיוטת הטופס נשמרת ב־`sessionStorage` לעשר דקות בלבד; תוכן ההודעה הרפואית אינו נשמר.
- חלון הטיפולים כולל כעת focus trap, סגירה ב־Escape, החזרת המיקוד לטריגר ונעילת גלילת רקע בטוחה.
- תפריט המובייל, FAQ וכלי הנגישות קיבלו סמנטיקת ARIA, התנהגות מקלדת ויעדי מגע של לפחות 44×44 פיקסלים.
- נוסף `prefers-reduced-motion`, סגנון `focus-visible` וקישור דילוג לתוכן.
- כתובות canonical, Open Graph, sitemap ו־robots משתמשות בכתובת GitHub Pages האמיתית וניתנות להחלפה דרך `NEXT_PUBLIC_SITE_URL`.
- Next.js, React ותלויות הבדיקה עודכנו. Next.js הוא `16.2.10`, מעל גרסאות התיקון של התראות האבטחה הרלוונטיות ממאי 2026.
- תמונות ממשק פעילות הוקטנו והומרו ל־WebP. פוטנציאל בזבוז התמונות לפי Lighthouse ירד מכ־239KiB לכ־22KiB.
- נוסף CI שמריץ lint, TypeScript, כיסוי בדיקות, Playwright, build ואימות export לפני פריסה.

## בדיקות שבוצעו

- `npm run check`: lint, TypeScript ו־14 בדיקות יחידה/אינטגרציה — עברו.
- כיסוי אוטומטי: 66.28% statements, ‏60.05% branches, ‏55.88% functions ו־68.04% lines. לוגיקת טופס יצירת הקשר מכוסה ב־100% שורות.
- `npm run test:e2e`: ‏36 בדיקות עברו ו־6 בדיקות mobile-only דולגו כמצופה בפרופילי desktop.
- מנועי דפדפן: Chromium, Firefox ו־WebKit/Safari.
- פרופילי מכשיר: מחשב, Pixel 7/Android, iPhone 13 ו־iPad Pro 11; נבדקו overflow, תפריט מגע, modal, FAQ, טופס וכלי נגישות.
- axe-core: אפס הפרות אוטומטיות של WCAG A/AA בכל ששת הפרופילים.
- build של Next.js: נתיבי האתר הסטטיים נוצרו בהצלחה; canonical, base path, assets, sitemap ו־robots אומתו תחת `/roei2`.
- `npm audit`: אפס פגיעויות ידועות.
- Lighthouse על export production: נגישות 100, Best Practices ‏100, SEO ‏100. שרת הבדיקה המקומי, שאינו דוחס HTML, קיבל 73 בביצועים; המדד הנצפה היה LCP כ־1.28 שניות, CLS ‏0 ו־TBT ‏178ms.

## פעולות חובה לפני עלייה לאוויר

1. להגדיר ב־GitHub Actions את `WEB3FORMS_ACCESS_KEY`. זהו מפתח client שנחשף ב־bundle מטבעו; אסור לשים בו סוד שרת או הרשאות נוספות.
2. לבצע בדיקה ידנית על מכשירים פיזיים עם VoiceOver ב־iPhone/iPad ו־TalkBack ב־Android. אמולציה ו־axe אינן תחליף לאישור נגישות ידני או משפטי.

## מגבלות והמלצות המשך

- GitHub Pages אינו מאפשר להגדיר מהפרויקט כותרות HTTP כגון CSP, HSTS ו־Permissions-Policy. אם נדרשות כותרות אבטחה קשיחות, יש לפרוס מאחורי CDN/אירוח שמאפשר אותן.
- השליחה מתבצעת ישירות מהדפדפן ל־Web3Forms. להגנה חזקה יותר מפני abuse, ולשליטה ב־validation, rate limiting ולוגים, מומלץ בעתיד endpoint שרת/Edge בבעלות האתר.
- כלי נגישות באתר הוא עזר בלבד ואינו מהווה הוכחת עמידה בתקן.
- יש לשקול בדיקת חדירה חיצונית ובדיקת נגישות אנושית לפני קמפיין רחב או קליטת מידע רגיש בהיקף משמעותי.
